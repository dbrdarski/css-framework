Project structure

The framework is organized in two folders, clientlib-framework and clientlib-framework-theme.

The clientlib-framework only defines mixins and variable default values. It does not output any CSS styles. This is an done intentionally, so it can be easily included and used in other AEM components.

Note: The clientlib-framework should be regarded as a project dependancy and should not be edited for any reason. The default values can be overriden into the theme folder. All the active development should take place into the framework-theme folder instead. The framework files should not be edited for any reason except for updating or improving the framework itself. This separation guarantees that any project customizations won't be lost during an framework upgrade.

Developing a theme

The theme folder is notable for two things:

  • The framework-theme outputs all necessary framework styles (CSS resets, utility classes, layout grids and default styles). This is in contrast to the framework which doesn't output any CSS when included.
  • When implementing a design within the framework, all project related customization and development should take place in the theme folder. This also includes default theme variables, if we want to change a default vlue it should be overriden form the theme folder.

Overriding default variables

Instead of editing the default variables in the framework folder, we need to override them into the theme-variables.less file in parent theme folder.

Including the framework in components

To get access to the framework from within an ACL component it must be included included on the top of the LESS file. Including the the framework does not output any CSS.

Media Mixins

Media mixins that allow for smart media queries that use the built-in screen breakpoints in variables.less. These values can be overriden in theme-variables.less.

.screen(sm-lg, {
   .title {
      color: #6f5499;
      font-weight: 900;
   }
});

outputs:

@media (min-width: 576px) and (max-width: 1199px) {
   .title {
      color: #6f5499;
      font-weight: 900;
   }
}

LESS Iterators

Less doesn't offer FOR loops and such, instead iteration is achived by calling mixins recursively. This can get confusing and complicated. These iterators mixins provide descriptive and more unified approach for working with Ranges, Lists, Hashmaps and Media (viewport sizes).

Range Iterator

Accepts a range and a ruleset containg a mixin. Applies the mixin to each number value from the range. If the first parameter of the list is ommited, it defaults to 1.

.range.map(@to, @block)

// will iterate numbers from 1 to 8 .range.map(8, { .fn(@i){ // @i will be the iterated value ... } });

.range.map(@from @to, @block)

// will iterate numbers from 10 to 20 .range.map(10 20, { .fn(@i){ // @i will be the iterated value ... } });

Example

.range.map(3, { .fn(@i){ .column-@{i}{ width: 100% / @i }; } });

The code will output this:

.column-1 { width: 100%; } .column-2 { width: 50%; } .column-3 { width: 33.33333333%; }

List Iterator

.range.map(@list, @block [, @modifiers])

.list.map(one two three, { .fn(@i){ // @i will be the iterated value ... } });

Example

.list.map(left right center justify, { .fn(@i){ .text-@{i}{ text-align: @i; } } });

Will output:

.text-left { text-align: left; } .text-right { text-align: right; } .text-center { text-align: center; } .text-justify { text-align: justify; }

Hash Map (Key/Value Pair) Iterator

.hash.map(key value, key value; @block)

Disclosure: There are no hashmap data type in LESS.
The way we emulate hashmaps is by nesting a 2 member list (instead of key and value) into another list.

.hash.map(k1 v1, k2 v2; { .fn(@k, @v){ // @k and @v will be the iterated key/value pairs ... } });

Example

@brand-colors: success #ecf0f1, warning #bdc3c7, info #2c3e50, danger #1abc9c; .hash.map(@brand-colors, { .fn(@k, @v){ .btn-@{k}{ background-color: @v; } } });

Will output:

.btn-success { background-color: #ecf0f1; } .btn-warning { background-color: #bdc3c7; } .btn-info { background-color: #2c3e50; } .btn-danger { background-color: #1abc9c; }

Media (Viewport Size) Iterator

.media.map(min/max, @block)

.media.map(min, { .fn(@s){ // @s will be the iterated screen size prefix ... } });

Example

.media.map(min, { .fn(@s){ .hide-@{s}{ display: none; } .show-@{s}{ display: block; } } });

Will output:

.hide-xs { display: none; } .show-xs { display: block; } @media (min-width: 576px) { .hide-sm { display: none; } .show-sm { display: block; } } @media (min-width: 768px) { .hide-md { display: none; } .show-md { display: block; } } @media (min-width: 992px) { .hide-lg { display: none; } .show-lg { display: block; } } @media (min-width: 1200px) { .hide-xl { display: none; } .show-xl { display: block; } }
