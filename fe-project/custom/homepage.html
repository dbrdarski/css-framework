ACL LESS Framework

Project structure

The framework is organized in two folders, clientlib-framework and clientlib-framework-theme.

All project-related development should take place in the theme folder. The framework files should not be edited for any reason except when working on improvements to the framework itself, or updating to a newer version. This structure guarantees that any project related customizations won't be deleted in the process.

Block Grid (based on ZURB Foundation)

Block grid is an alternative grid system used when we want to display blocks of even width. Appart from not being able to have 5 identical columns with the 12 columns grid system, the block grid also adds clear:both to elements that start in new line. In the example below, the blue boxes are 1px shorter than the rest. This causes broken layout. Switch to a tablet viewport mode to view the the broken layout!

Media Mixins

Media mixins that allow for smart media queries that use the built-in screen breakpoints in variables.less. These values can be overriden in theme-variables.less.

.screen(sm-lg, {
   .title {
      color: #6f5499;
      font-weight: 900;
   }
});

outputs:

@media (min-width: 576px) and (max-width: 1199px) {
   .title {
      color: #6f5499;
      font-weight: 900;
   }
}

LESS Iterators

Less doesn't offer FOR loops and such, instead iteration is achived with recursive mixins. This can get confusing and complicated.

These iterators mixins provide descriptive and more unified approach for working with Ranges, Lists and Media.

Range Iterator

.range.map(@block, @to)

.range.map({
   .fn(@i){
      // @i is the iterated value;
      ...
   }
}, 8);

.range.map(@block, @from, @to)

List Iterator

.range.map(@list, @block [, @modifiers])

Media Iterator

.range.map(min/max, @block)
